Level 2 — Configure Your Base Camp  +100 XP

The one-time VS Code setup that makes every future prompt smarter

Most people skip this step and wonder why Copilot keeps generating prod-storage-account-01 instead of their naming convention. The fix is a single file that pays back every day.

The power move: Create .github/copilot-instructions.md in every IaC repository. This file is injected into every Copilot prompt automatically — you never have to repeat yourself.

❌ Without instructions file
resource "azurerm_key_vault" "main" { name = "my-keyvault" sku_name = "standard" # purge protection? missing # soft delete? not set # public network? open # access policy model }
✅ With instructions file
resource "azurerm_key_vault" "prod-eus2-kv-01" { name = "prod-eus2-kv-01" sku_name = "standard" purge_protection_enabled = true soft_delete_retention_days = 90 public_network_access_enabled = false enable_rbac_authorization = true }
📄 .github/copilot-instructions.md 
## Project Context
Terraform repo for [Company] Azure landing zone.

## Naming Convention
{env}-{region-short}-{service}-{instance}
Example: prod-eus2-kv-01
Regions: eastus2=eus2, westus2=wus2

## Non-negotiable Defaults
- Provider: azurerm >= 3.90.0
- Tags: Environment, Owner, CostCenter, Project (always)
- Key Vault: purge_protection=true, soft_delete=90, RBAC model
- Storage: min TLS 1.2, public_blob_access=false
- No hardcoded secrets

## Repo Structure
modules/  environments/prod/  environments/nonprod/
💡 Pro Tip

Install these 4 extensions alongside Copilot: HashiCorp Terraform, Azure Bicep, YAML (Red Hat), Azure Tools. They give Copilot schema awareness — completions become syntactically correct, not just plausible.

Level 3 — Master the Prompt Duel  +100 XP

Side-by-side: weak vs. expert prompts for IaC

The difference between a useless AI output and a production-ready module is almost entirely in the prompt. The formula: Role → Context → Task → Constraints. Four fields. That’s it.

❌ Weak Prompt
✅ Expert Prompt
write terraform for azure key vault
You are a senior Azure Terraform engineer. Create a reusable module for Azure Key Vault. - azurerm >= 3.90.0 - Inputs: name, rg, location, tenant_id - Purge protection on, soft-delete 90d - Public network access disabled - RBAC authorization (not access policies) - Diagnostics to var logAnalyticsWorkspaceId - Standard tag map merged with locals Outputs: key_vault_id, key_vault_uri Explain each security choice inline.
💡 The weak prompt generates a skeleton you’ll rewrite. The expert prompt generates a production module you’ll spend 10 minutes reviewing.
❌ Weak Prompt
✅ Expert Prompt
check this bicep for security issues
Review the following Bicep against Azure Security Benchmark v3. Focus: network exposure, identity, encryption at rest, logging, secrets. For each issue: state the control ID (e.g. NS-1, DP-3), describe the risk, provide the corrected Bicep snippet. [paste your Bicep here]
💡 Asking for control IDs maps directly to your compliance docs. Review AND audit evidence in one shot.
❌ Weak Prompt
✅ Expert Prompt
write an ADR for using API Management
Write an ADR in MADR format. Decision: Azure APIM as API gateway for all Azure OpenAI deployments. Context: 12 teams, need token quota per team, PII detection, centralised logs, failover across 2 PTU slots. Alternatives: Front Door only, direct endpoints, NGINX custom GW. Include: pros/cons table, decision outcome, consequences.
💡 ADRs are high-value and tedious. Giving actual context produces a first draft that is 80% done.

Level 4 — Cheat Codes  +100 XP

Append these to any prompt for instant quality upgrades

These are prompt suffixes that take 3 seconds to type and transform output quality. Save them in your prompt library today.

Explain Reasoning
“...explain each security choice inline.”
Adds inline comments justifying every non-obvious decision. Makes review 5x faster.
Surface Hidden Assumptions
“What assumptions did you make?”
Ask after any response. Catches silent choices like “I assumed azurerm 4.x” before they break your pipeline.
Failure Mode Scan
“What are the failure modes of this design?”
Forces the model to surface edge cases it was aware of but didn’t mention. Gold for architecture reviews.
Dry-Run Summary
“Summarise what terraform plan would show: created, destroyed, updated.”
Sanity check before running actual plan. Catches accidental resource replacements early.
CIS Audit Pass
“Review against CIS Azure Benchmark 2.0. List non-compliant settings.”
Instant security posture check on generated AND inherited code.
Test Generator
“Generate a Terraform native test that validates [specific property].”
High-leverage, low-risk. Gets your module test coverage from zero to something in minutes.

Level 5 — Boss Challenges  +100 XP

The four scenarios where AI will confidently lead you off a cliff

These are not theoretical risks. They show up in real enterprise PRs every week. Know them before you hit them.

🔥 Boss 1 — The Open Door

AI-generated IAM and firewall rules

AI models are trained on public GitHub code, full of * wildcards, 0.0.0.0/0 CIDR blocks, and Contributor roles at subscription scope. The model produces what it has seen most often.

  • Never merge AI-generated IAM or firewall rules without a dedicated security review
  • State least-privilege in your prompt: “Use minimum necessary permissions. No wildcard actions or overly broad scopes.”
  • Run Checkov or tfsec as a PR gate — catches the most common patterns automatically
🔥 Boss 2 — The Secret Leak

Pasting credentials into the prompt window

A developer is debugging at 6pm, pastes a full config block including the password into Copilot Chat. Even on enterprise plans with data handling agreements, this is a security violation and a habit that spreads.

  • Never paste secrets, connection strings, or API keys into any AI assistant
  • Replace sensitive values with [REDACTED] or variable placeholders before pasting
  • Add *.env, *secrets*, *credentials* to Copilot Enterprise content exclusion patterns
🔥 Boss 3 — The Stale API

Provider API version drift

AI training data has a cutoff. The azurerm provider releases minor versions every 2–3 weeks. The model may generate deprecated arguments or miss new required properties.

  • Always specify the exact provider version: “using azurerm provider 3.90.0”
  • Run terraform validate immediately — schema errors appear before you waste time on a plan
  • Pin provider versions in versions.tf and upgrade deliberately, not automatically
🔥 Boss 4 — The Confident Hallucination

Plausible-but-wrong resource properties

The model will invent property names that look exactly right — enable_encryption, private_access_enabled — that do not exist in the provider schema. The code fails terraform validate, but only if you run it.

  • For any resource type you haven’t personally used: cross-check against the Terraform registry docs
  • Always run terraform validate before terraform plan — always. This is the minimum bar.
  • Use /explain on unfamiliar provider resources before generating code that uses them

Level 6 — Enterprise Governance Mode  +150 XP

Controls you need before scaling AI across your organisation

Individual productivity is easy. Governing AI usage across 20 teams, mixed compliance requirements, and varied skill levels — that’s where most enterprises stumble.

RiskSeverityMitigation
Secrets in prompts
Devs paste prod creds into Copilot Chat		 High Content exclusions in Copilot Enterprise + developer training + DLP on IDE traffic
AI-generated misconfigurations
Open ports, permissive IAM, disabled encryption		 High Checkov / tfsec / Defender IaC scanning as mandatory PR gates — AI code is not exempt
Blind trust
Team skips review because “AI wrote it”		 High PR label “AI-assisted” triggers security review for IAM, network, encryption files
IP / license contamination
Copilot reproduces GPL’d public code		 Medium Enable Copilot code-referencing filter (Enterprise) — blocks suggestions matching restrictive licenses
Data residency
Prompts sent to shared inference infrastructure		 Medium Azure AI Foundry private deployment for regulated workloads. Confirm Copilot DPA covers your jurisdiction.
Skill atrophy
Engineers stop understanding what they ship		 Medium Require engineers to explain AI-generated code during PR review. Accelerate, don’t replace understanding.

The 3-control starter pack for any organisation:

  1. Checkov in every IaC PR gate — catches the top 20 AI-generated misconfiguration patterns before a human reviews
  2. One-page “What not to do” guide — what not to paste, how to review security-critical outputs, when to escalate. In onboarding, not buried in a wiki.
  3. Shared prompt library — a versioned repo of proven prompts. New team members start effective on day one.
