About Me

Azure Cloud Solutions Architect (AZ-305)  ·  Hands-On across Infrastructure, DevOps & Security  ·  21+ Years  ·  8+ on Microsoft Azure

Azure Cloud Solutions Architect (AZ-305) with 21+ years across enterprise IT, hybrid cloud, and information security — including 8+ years architecting and operating Microsoft Azure platforms. Designs target-state cloud architectures and Azure landing zones; defines the standards for Infrastructure-as-Code (IaC), DevSecOps, networking, and security baselines; and governs identity (IAM / Zero Trust), observability / SIEM, and business continuity / disaster recovery (BCDR) across multi-customer SaaS environments.

A hands-on architect — codes the Terraform, tunes the WAF, and writes the KQL, not just the diagrams. Architecture decisions proven against HIPAA, HITRUST, ISO 27001, and SOC 2, with hands-on leadership of certification programmes and Information Security Operations (SOC, SIEM, NGFW, DLP, PAM). Career spans datacenter modernisation, Microsoft 365 and Azure cloud adoption, hybrid identity (Active Directory + Microsoft Entra ID), and large-scale infrastructure / cloud operations across Azure, GCP, and on-prem estates.

Architecture & Engineering

Design Decisions I Own — and Build

Cloud Architecture & IaC: Design Azure landing zones and hub-spoke reference architectures with subscription / management-group topology and policy guardrails — then codify them as reusable Terraform modules and Bicep templates (parameterised, source-controlled, PR-reviewed).

DevSecOps & Release Engineering: Define CI / CD and DevSecOps standards — implemented as Azure DevOps YAML pipelines with SonarQube quality gates, branch policies, environment approvals, and GitOps-style promotion.

Network & Edge Security Architecture: Architect secure ingress and segmentation — Front Door + Application Gateway WAF (OWASP CRS + custom rules, exclusions, geo-filtering), Azure Firewall, and Private Link / Private Endpoint topology — and build the rule sets myself.

Identity & Zero Trust: Define the Zero Trust identity model — Microsoft Entra ID RBAC, Privileged Identity Management (PIM), Conditional Access (sign-in risk, named locations, device compliance), and Azure AD B2C — and configure it end to end.

Observability & SecOps: Establish monitoring and detection standards — Application Insights metrics, Azure Monitor alert rules, Log Analytics KQL, and Microsoft Sentinel analytics / hunting rules.

Resilience & BCDR: Design BCDR architecture with defined RPO / RTO — Azure Backup policies, Azure Site Recovery (ASR) replication, and Key Vault secret rotation — validated through recovery drills.
